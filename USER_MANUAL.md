# 智慧树网课自动化辅助脚本 — 用户操作手册

## 免责声明

> 本脚本仅用于**自动播放视频**和**自动处理不计分的课堂互动**，不自动完成任何计分测试、考试或作业。请合理使用，遵守所在院校的学术规范。

---

## 一、快速开始

### 1.1 安装 Node.js

如果你还没有安装 Node.js：

1. 访问 https://nodejs.org/ 下载 LTS 版本（左侧按钮）
2. 安装时一路点"下一步"即可
3. 安装完成后打开命令提示符 (CMD) 或 PowerShell，运行以下命令验证：

```bash
node --version
npm --version
```

能看到版本号就说明安装成功。

### 1.2 安装依赖

进入项目目录，安装所需的库：

```bash
# 进入项目目录
cd zhihuishu-automation

# 安装依赖（Playwright 等）
npm install

# 安装 Chromium 浏览器（Playwright 需要）
npx playwright install chromium
```

> npx playwright install chromium 会下载约 200MB 的 Chromium 浏览器，请确保网络通畅。

## 二、使用方式（两种）

### 方式 A：默认模式（推荐）

脚本启动**自带 Chromium 浏览器**，和你的日常浏览器完全隔离。

```bash
npm start
```

**步骤：**
1. 脚本会自动打开一个 Chromium 浏览器窗口，进入智慧树首页
2. **在窗口中手动登录**智慧树账号
3. 进入课程，点击任意**视频**开始播放
4. 脚本检测到视频后自动接管，**浮窗控制面板**出现在页面右下角

### 方式 B：附加到 Edge（--attach）⭐ 推荐

用你电脑上的 Edge 浏览器，脚本**自动启动 Edge 并连接**。

```bash
npm start -- --attach
```

**详细步骤：**

**第 1 步：打开终端**

在项目目录下右键 → "在终端中打开"，或者先 `cd` 到项目目录。

**第 2 步：运行命令**

```bash
npm start -- --attach
```

你会看到终端开始输出日志。

**第 3 步：观察 Edge 自动启动**

脚本会自动：
1. 杀掉后台残留的 Edge 进程（终端会显示相应日志）
2. 等 1~2 秒后，**自动弹出一个新的 Edge 窗口**
3. 这个 Edge 直接打开了智慧树首页

> 弹出的 Edge 用的是**临时配置**，和你平时用的 Edge 是隔离的，互不影响。
> 你的书签、历史记录、已安装的扩展都不会出现在这个窗口里。

**第 4 步：登录智慧树**

在弹出的 Edge 窗口中：
1. 点击右上角 **"登录"**
2. 选择 **"密码登录"** 或 **"短信登录"**
3. 输入你的账号密码，完成登录

**第 5 步：进入课程播放视频**

1. 登录后，在智慧树首页点击 **"我的学堂"**
2. 找到你的课程，点击 **"继续学习"**
3. 在课程目录中，点击任意一个**视频标题**
4. 等待视频播放器加载完成

**第 6 步：脚本自动接管**

当你打开视频后，脚本会自动检测到视频播放器：
- 终端会显示 `检测到视频页面！`
- 智慧树页面**右下角**会出现一个深色**浮窗控制面板**
- 视频自动以 **1.5 倍速** 开始播放

**第 7 步：挂机即可**

此时你不需要做任何操作了：
- ✅ 视频自动播放
- ✅ 课堂互动自动回答
- ✅ 视频结束自动切换下一节
- ❌ 单元检测自动跳过（留给你自己做）

你可以：
- 把 Edge 窗口**最小化**（视频继续播放）
- 把窗口移到**后台**，做其他事
- 或者用 `--headless` 模式连窗口都不显示

**第 8 步：退出脚本**

在终端按 `Ctrl + C` 即可退出。
退出时脚本断开连接，**不会关闭 Edge 窗口**，你可以继续手动浏览。

## 三、命令行选项

```bash
npm start                  # 默认模式：启动自有 Chromium
npm start -- --attach      # 附加到本机 Edge/Chrome（自动启动）
npm start -- --headless    # 无窗口后台运行（完全静默）
npm start -- --debug       # 调试模式（更多日志）
npm start -- --attach --headless   # 附加到 Edge + 后台运行
```

**组合使用：**
```bash
# 后台附加到 Edge，最省心
npm start -- --attach --headless --debug
```

### 界面主题
- **背景色**：`#0b1214` 深青底色，93% 不透明度
- **边框**：2px 细边框 `#09363d`，微磨砂玻璃质感（`backdrop-filter: blur`）
- **字体**：`#9ae8f1` 青色，干净清晰

- **不影响智慧树** — 浮窗只显示状态，不干扰页面功能

## 五、自动化流程

脚本检测到视频后，自动执行以下操作：

| 环节 | 行为 |
|------|------|
| 视频播放 | 以 **1.5 倍速** 自动播放 |
| 暂停检测 | 如果视频暂停，自动恢复播放 |
| 卡顿检测 | 当前时间 25 秒无变化 → 自动恢复或刷新页面 |
| 课堂互动 | 弹出互动题时 → 等 3 秒 → 随机 1 或 3 次点击（避免 2 次点掉选项）→ 每点间隔 1 秒 → 2 秒后点关闭 |
| 继续学习 | 检测到"继续学习"弹窗 → 自动点击 |
| 下一节 | 视频结束 → 自动查找并点击"下一节" |
| 单元检测 | 检测到真正的测试页面（有题目 + 提交按钮）→ 暂停自动化，等你手动完成 |
| 页面崩溃 | 自动刷新，恢复运行 |

> **注意：** 单元检测/考试页面只有出现**大量题目**（多道选择题、填空题等）才会被识别。侧边栏或目录中的"单元检测"文字不会触发误判。

## 六、反自动化检测

智慧树有反脚本检测机制，脚本内置了多层对抗：

| 检测项 | 脚本处理 |
|--------|---------|
| `navigator.webdriver` | 改为 `undefined`（直接在实例上覆盖，最关键） |
| document webdriver 属性 | 移除 + MutationObserver 防止恢复 |
| `window.chrome.runtime` | 补全完整 API |
| `chrome.csi / loadTimes` | 补全（真实 Chrome 有这些） |
| `navigator.plugins` | 填充 3 个真实插件（长度不为 0） |
| `navigator.languages` | 设为 `['zh-CN','zh','en']` |
| `navigator.deviceMemory` | 设为 8GB |
| `navigator.hardwareConcurrency` | 设为 8 核 |
| `$cdc_ / $chrome_` 痕迹变量 | 启动时清除 |
| 可见性检测 | 覆盖 `hidden` / `visibilityState`，最小化也不暂停 |
| 自动化标记 | `--disable-blink-features=AutomationControlled` |

反检测脚本在**页面任何脚本执行之前**注入（自有 Chromium 模式），智慧树无法察觉。

## 七、答题流程说明

### 点击策略

防止反选导致无法关闭：

```
① 等待 3 秒（模拟真人审题）
② 随机决定：点 1 次 或 3 次（概率各 50%）
   → 跳过 2 次，避免第二次把已选中的选项点掉
③ 每次点击间隔 1 秒
④ 选项完成后 2 秒，点击关闭按钮
```

### 点击效果

每次点击选项时，被点击位置会出现**蓝色扩散圆环**动画，让你直观地看到脚本点在了哪里：

```
  ① 鼠标缓慢移动到目标附近
  ② 精确定位到目标
  ③ ● ← 扩散圆环（0.6 秒后消失）
  ④ 点击
  ⑤ 点完后微移（模拟手指抬起）
```

这个效果只在点击互动题时出现，不影响智慧树页面本身。

## 九、常见问题

### Q1: 一启动就被识别为异常脚本？

脚本已内置多层反检测机制。如果仍然被检测：
1. 检查是否使用了 `--attach` 模式（推荐）
2. 用 `--debug` 模式运行查看日志

### Q2: 视频页面比例奇怪？

脚本使用 1920×1080 视口。如果仍然异常，尝试手动调整窗口大小。

### Q3: 找不到"下一节"按钮？

智慧树界面更新后按钮文本可能变化。打开 `config.js`，在 `buttonTexts.next` 数组中添加新的文本。

### Q4: 互动题没有被处理？

1. 确认是"课堂互动"（不计分类型），而不是"单元检测"
2. 运行 `npm start -- --debug` 查看日志
3. 检查 `config.js` 中 `interactionKeywords` 是否匹配

### Q5: 脚本卡住了怎么办？

1. 等待 30 秒，脚本会自动检测卡顿并恢复
2. 如果长时间无响应，手动刷新智慧树页面
3. 或者关闭脚本 + 浏览器，重新运行

### Q6: 最小化后视频会停吗？

加了可见性覆盖，**最小化也不会暂停**。也可以用 `--headless` 模式完全后台运行。

### Q7: 浏览器窗口能关掉吗？

不能关掉自动化窗口。但可以：
- 把窗口**最小化**（视频继续播放）
- 把窗口移到**另一个虚拟桌面**
- 用 `--headless` 模式完全后台运行（不显示窗口）

### Q8: 点击暂停后计时器还在走吗？

**不再走了。** 点击暂停按钮后，计时器同步冻结，恢复时继续累加。计时器只统计**实际运行时间**。

### Q9: 折叠后再展开面板布局乱了吗？

已修复。主内容统一包裹在容器内，折叠/展开只切换整个容器，不会影响内部的 flex/grid 布局。

## 十三、技术原理

### iframe 穿透

```
主页面 (study.zhihuishu.com)
  └── iframe → iframe → <video>
```

`page.frames()` 获取所有 frame，逐个查找 `<video>`。

### 计时器

浏览器端独立 `setTimeout`（500ms 间隔），使用**累积时间**而非 wall clock。暂停时只冻结累加，恢复后继续。重新注入面板时先停止旧计时器，防止新旧计时器冲突导致时间跳跃。

### 单元检测智能识别

不再仅靠关键词匹配。检测流程：
1. 有可见视频 → 不是测试页
2. 关键词出现 3 次以上 → 继续判断
3. 页面有 2+ 个 radio/checkbox → 是测试页
4. 有关键词 + 提交按钮 → 是测试页
5. 否则 → 不是测试页（避免侧边栏文字误判）

### 反检测

`context.addInitScript()` 在页面任何 JS 前注入，覆盖所有自动化检测点。

### 智能按钮识别

通过文本内容匹配（不依赖 class 名），支持 `getByRole`、`getByText`、XPath 三种策略。
